Algoritimos Ordenação

Algoritmo de ordenação é um conjunto instruções para ordenar um array. Podemos também ordenar mais do que tipos númericos, podemos ordenar qualquer tipo de dado, se implementarmos uma codição de comparação ao tipo.

Propriedades Importantes

Algoritmo Estável
Preserva a ordem dos elementos
Algoritmos in-place
Não usa memória extra

Dividimos os Algoritmos de Ordenação por:

  • Por comparação
    • Simples
    • Dividir para conquistar
  • Em tempo linar
    • Preço a pagar (Memória)

SelectionSort

É o algoritmo mais intuitivo, provalvelmente se você já teve que fazer um, ele deve ter sido o selection sort.

Ideia principal do algoritmo

Percorre o array para descobrir qual é o menor elemento comparando com cada elemento no array, assim caso encontre um elemento que seja menor do que o primeiro, que de início é considerado o menor, então ele passa a ser o menor, ate que se encontre outro, caso não seja encontrado outro, ele é o menor elemento, assim ao final dessa execução o menor elemento encontrado vai assumir a primeira posição do array,

Usando a ideia principal para ordenar um array

Para ordenar um array, executamos essa ideia principal n vezes, sendo que em cada uma das vezes o primeiro elemento vai ser outro, sempre sendo o próximo elemento.

InsertionSort

Tem como rotina base a inserção ordenada, executamos diversas vezes essa rotina, n - 1 vezes

Inserção ordenada

Temos um array qualquer, o qual temos que ordenar um elemento que não esta na primeira posição, assim comparamos esse elemento com elementos anteriores dele e se ele for menor, então trocamos ate algum elemento ser menor que ele, ou ele atingir o primeiro indice do array.

Usando a inserção para criar o insertionSort

Tendo em mente a inserção ordenada, basta aplicar ela diversas vezes, porém vamos começar comparando da segunda posição, de forma que cada vez vamos usar um indice maior para ser comparando com seu antecesor.

MergeSort

É um algoritmo eficiente de ordenação por divisão e conquista, independentemente da organização do array o tempo que ele leva para ordenena-lo é Θ(n*logn).

O funcionamento do mergeSort basea-se em uma rotina chamada merge

Merge

É uma rotina que combina dois arrays ordenados em um outro também ordenado, porém não vamos criar um novo array, pois isso requer transferir os dados e isso custa memória e processamento, ao invés disso fazemos o merge de duas partes ordenadas de um array, não um merge de dois arrays.

O merge recebe o array, um indice left, right e middle, assim fazemos um array auxiliar, que vai guardar a inteira parte a ser organizada, ou seja, do left até o right. Depois definimos três indices que serão onde começa a primeira parte a ser organizada, i = left, a parte que termina essa primeira parte é o próprio middle, o indice que começa a primeira parte, j = middle + 1, a parte que termina é o próprio right, e a última variável é a qual vai representar a respectiva posição que está sendo organizada, k = left, lembrando que os indices que começam as partes serão os que vão ser incrementados a cada vez que um valor, a qual ele é indice do array auxiliar, i ou j, representa, for maior que o outro, ou seja quando compararmos no array auxiliar: array[i] > array[j], então o array indice j vai ser incrementado, caso o array[i] seja menor, então ele vai ser incrementado além de que o valor que ocupar essa posição de menor vai ser colocado no array que está sendo ordenado e o indice que representa a respecitiva posição desse array, k, vai ser incrementado também.

Ao termino dessa execução, em muitos casos pode acabar por sobrar dados a serem incluidos do array auxiliar para o originial em uma das partes, sendo assim precisamos colocar eles no array que esta sendo ordenado, para isso enquanto a váriável que está menor que o limite de sua parte, i ou j, será colocado no array que está sendo ordenado, e cada um dos indices será incrementado.

Usando a ideia de merge para criar o MergeSort

Recursivamente divimos o array em dois, até que em cada uma das duas partes tenha apenas dois elementos, precisamos também que tudo isso seja feito na recursão, de modo que o merge seja aplicado primeiro nas partes menores, só assim é possível fazer a ordenação.
